# Week 10 — Resources

Every link below is free to read, free to bookmark, and free to cite. The track ends here; the bookmark file is the one that should outlive the cohort. Treat the *organizer guides* and the *code-of-conduct sources* as canonical references — re-read the linked sections each time you scope a new event.

## The organizer canon (read first, re-read often)

- **Major League Hacking (MLH) Organizer Guide** — `https://organize.mlh.io/` — the most-cited organizer reference in the student-hackathon ecosystem. The membership tiers (Local, Member, Premier), the application timeline, the hardware lab, the swag program, the insurance offering, the judge introductions, the marketing reach. Even if your event is not MLH-affiliated, the guide is the floor for what a well-run hackathon looks like.
- **MLH Code of Conduct** — `https://mlh.io/code-of-conduct` — the most adopted code of conduct in the student hackathon space. Permissively licensed for adoption with attribution. Pair with the *incident response chain* (Lecture 3 §4).
- **NumFOCUS Event Playbook** — `https://numfocus.org/programs/events` — the open-source-event playbook that complements MLH's hackathon-specific guide. Strong on budgeting, sponsorship tiers, accessibility, and post-event reporting.
- **GitHub Education — Student Hackathon Resources** — `https://education.github.com/teachers` — the GitHub side of student events. The Student Developer Pack is the recurring sponsorship-adjacent ask (every attendee can claim free credits).
- **Devpost Help Center — Organizers** — `https://help.devpost.com/hc/en-us/categories/360001928711-Organizers` — the platform-side documentation for the most-used hackathon submission platform. Setting up a hackathon page, judge accounts, rubrics, sponsor logos, prize listings.

## Insurance, food, dietary (the boring stuff that matters)

- **MLH Member tier — Insurance benefit** — described inside the Organizer Guide (membership tiers section). Member-tier MLH events get event liability coverage through MLH's group policy.
- **University Risk Management offices.** Each university has a Risk Management office that can either insure the event under the university umbrella or refuse to. The named *who-signs* question (Lecture 2 §5) is answered there.
- **The Eventgenic insurance product overview (for context)** — `https://www.eventgenic.com/insurance` — third-party event-insurance market overview; do not endorse a single provider; useful for understanding the *typical coverage levels*.
- **USDA — Food allergen labeling** — `https://www.usda.gov/topics/food-safety/food-labels` — the regulatory floor for food labeling in the US. Student events are not commercial, but the labeling conventions are still expected by attendees.
- **AllergyEats / SpokinApp public lists** — directory-style references for *what allergies are most common*. Useful for the dietary checklist (Lecture 2 §6).

## A short editorial on bookmarks

Most of these links will outlive the cohort. Bookmark them. Some of them — MLH Organizer Guide, NumFOCUS Event Playbook, MLH Code of Conduct, the Devpost organizer help — should be in a *single folder* in the learner's bookmark bar called `hackathon-organizing/`. The folder is the thing the learner shares with the next co-organiser. The folder is the thing the learner re-opens 18 months from now when the next event is on the calendar. Curate it now.

## A note on retired links

Three categories of link in this resources file will rot fastest:

- **Annual recap pages** at HackMIT, TreeHacks, PennApps, HackTX, BoilerMake, Hack the North. These pages typically rotate each year — the *current* recap is on the front page, prior years are archived under `/past` or a similar slug. Bookmark the *event organisation's homepage*, not the specific-year recap URL, to follow forward.
- **MLH tier benefits and fee structures.** MLH has revised tier benefits at least twice in the C4 era (Local being free, Member adding insurance, Premier adding dedicated staff). Verify the current tier benefits with MLH at the time of your application, not against this file.
- **Sponsor-program landing pages.** DigitalOcean Hatch, Twilio Quest, GitHub Education's event policies, NumFOCUS sponsorship — these rotate slug and re-brand. If a link returns a 404, search the sponsor's main site for *"student events sponsorship"* or *"hackathon sponsorship"* — the program still exists; the URL moved.

The bookmark folder should treat these three categories as *re-verify before each event*, not *one-time-bookmark-and-trust*.

## Reading order for a first-event organiser

If you have only 4–5 hours to read this whole list before scoping your first event, read in this order:

1. **MLH Organizer Guide** — full read, end-to-end. 90 minutes. The single highest-leverage document in the list.
2. **NumFOCUS Event Playbook — post-event reporting + sponsorship sections only.** 30 minutes. The named C4 *most-actionable* prose on these two topics.
3. **MLH Code of Conduct + the Berlin Code of Conduct.** 20 minutes. Read both, side by side, pick one to adapt.
4. **Two public retros** — pick one from HackMIT/TreeHacks/PennApps and one from BoilerMake/HackTX/Hack the North. 30 minutes total. Compare the *what they reported* lens against the *what they did not report* lens.
5. **The Devpost organizer help center — pages on setting up the event and on judge accounts.** 20 minutes. Practical mechanics for the platform most events use.

That sequence is the named C4 *first-event-organiser primer*. The remaining links in the file are *reference*, to be consulted as gates come up rather than read end-to-end.

## Reading order for an experienced organiser doing a re-up

If you've already run an event and are scoping the second one, the C4 voice is different. Skip the organiser guide re-read; you have the gist. Go to:

1. **Last year's own post-event close-out doc** (the `lessons-learned` paragraph from your `POST-EVENT-CLOSE-OUT.md` in Lecture 3 §6).
2. **The MLH membership status page** for any tier-change consideration.
3. **One peer-event retro published since your last event.** What did they try that you did not? What did they retire that you still do?
4. **The current sponsor-program landing pages** for the named recurring sponsors you partnered with last year — verify the tier benefits have not changed.

The re-up is faster than the first event by approximately 40 percent in scoping time, because the venue, the advisor, and the team are usually carry-over. The named *re-up gain* is sponsor and judge familiarity; year-two sponsors and judges convert at 2-3x the year-one rate.

## On the *no-paid-resources* policy

Every resource above is free. The C4 voice is firm on this: a first-event student team should not be paying for *organising resources*. The named paid-resources industry — paid courses on event-organising, paid templates for sponsor decks, paid contract-review services — is *not necessary* for a first event. The free organiser canon (MLH + NumFOCUS + the public retros + the public code-of-conduct sources) is the floor *and* the ceiling for what a first-event student team needs. Resist the temptation to subscribe to a $99 organiser-resource service; the value is in the *reading* of the free resources, not in the *purchasing* of paid ones.

The named paid line items in a first-event budget are *real-world expenses* — food, venue (if not free), AV rental, swag, insurance — not *organising resources*. Keep the line clean.
