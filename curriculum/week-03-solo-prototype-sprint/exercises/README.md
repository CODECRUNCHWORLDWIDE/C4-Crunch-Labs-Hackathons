# Week 3 — Exercises

Three short pre-build drills, done Tuesday through Friday. None of them write feature code — they prepare the artifacts you bring to the Saturday six-hour build. Skipping them is the single best way to fail the build.

## Index

1. **[Exercise 1 — Pick a real prompt](exercise-01-pick-a-real-prompt.md)** — write the one-sentence prompt your solo build answers. (~45 min, Tuesday)
2. **[Exercise 2 — The six-hour clock](exercise-02-the-six-hour-clock.md)** — pre-build the MoSCoW chart and the hourly clock on paper. (~60 min, Wednesday)
3. **[Exercise 3 — The three-minute demo](exercise-03-the-three-minute-demo.md)** — draft the demo script before you write the code. (~45 min, Friday)

## How to work the exercises

- Read the prompt. Skim, do not memorize.
- **Type or write the answer yourself.** Do not paste a giant snippet and edit. The point is muscle memory.
- Use the C4 voice: team-room, blameless, scope-discipline-first. No "rockstar" / "ninja" / "10x." Even when you are alone.
- Commit each completed exercise to the *Week 3 prep* folder of your `hackathon-template` repo (or a new `c4-week-03-prep-<yourhandle>` repo). Small commits.
- If you get stuck for more than 15 minutes, re-read Lecture 1 §2–§4, *then* try again.

These exercises are the difference between a six-hour build that ships and a six-hour build that flails. Do not skip them.
