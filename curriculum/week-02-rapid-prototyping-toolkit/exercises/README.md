# Week 2 — Exercises

Short, focused drills. Each one should take 45–90 minutes. Do them in order — later ones reference earlier ones.

## Index

1. **[Exercise 1 — Stand up the template](exercise-01-stand-up-the-template.md)** — create your `hackathon-template` repo on GitHub with the file shape from Lecture 2. (~60 min)
2. **[Exercise 2 — One-command spin-up](exercise-02-one-command-spinup.md)** — write and test `./bootstrap.sh new-event-name`. (~60 min)
3. **[Exercise 3 — Deploy to staging](exercise-03-deploy-to-staging.md)** — wire Vercel + Railway so the first push to a new event repo returns a live URL. (~60 min)

## How to work the exercises

- Read the prompt. Skim, do not memorize.
- **Type the answer yourself.** Do not paste a giant snippet and edit. The point is muscle memory.
- Use the C4 voice in any commit messages and README copy — team-room, scope-discipline-first, blameless. No "rockstar" / "ninja" language.
- Commit each completed exercise as you go. Small commits.
- If you get stuck for more than 15 minutes, skim Lecture 1 or 2 again, *then* search.

These exercises produce a real artifact you will use at every event. Treat them as work, not homework.
